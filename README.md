# AI-Native Marketing and Design Playbook

A working set of build rules for shipping products that are polished, that convert, and that do not look AI-generated. I run these as skills with Claude Code. They fuse the two things most people keep in separate teams: direct-response marketing (Cialdini, LF8, Hooked, growth loops) and real product and design engineering.

If you build with AI and the output keeps coming out generic, purple, and forgettable, this is the fix.

## What is inside

- **[01 Master build prompt](01-master-build-prompt.md)** — the base. Stack defaults, reference apps to match, Refactoring-UI rules, conversion (Cialdini and LF8), retention (Hooked), and a build process that ends in a real self-critique and score.
- **[02 Anti-AI-look](02-anti-ai-look.md)** — overrides the defaults. The banned "AI tells" (no purple, no Inter, no glassmorphism, no emoji bullets, no centered sameness) and the process that escapes the average: art direction first, explore three real directions, then pin an exact spec.
- **[03 Growth loops](03-growth-loops.md)** — replaces the "network effects" hand-waving. Design one closed four-stage loop baked into the core product, name its k-factor lever and cycle time.
- **[04 Gamification](04-gamification.md)** — the motivation architecture. Octalysis, Self-Determination Theory, player types, the mechanics library, and the line between borrowing casino mechanics and shipping dark patterns.
- **[05 Gamified without the tells](05-gamified-adhd-without-tells.md)** — the visual execution of gamification. How to make a tool feel fun, ADHD-friendly, and rewarding without tripping the banned list. Calm canvas, loud moments.
- **[06 Sales / CRM / outreach UI](06-sales-crm-ui.md)** — a domain pack for any pipeline or lead tool, with source-verified colors and patterns from Linear, Attio, Superhuman, HubSpot, Pipedrive, and the cold-email tools.

## How to use them

Drop these into your Claude Code project as skills, or paste the relevant one at the start of a build. They stack, and later files override earlier ones. Whatever you build, start with the direction step in `02` before writing a line of code. Every choice should have a stated reason. If a choice has no reason, it is a default, so replace it.
